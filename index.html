¿Qué es una Plataforma de Orquestación?

Una plataforma de orquestación de pipelines es un software que permite definir, programar, ejecutar y monitorear flujos de trabajo complejos que integran múltiples tareas y sistemas.

Definición

Especificar workflows usando código o interfaces visuales

Programación

Ejecutar tareas según cronogramas o eventos

Ejecución

Gestionar la ejecución distribuida de tareas

Monitoreo

Observar estado, logs y métricas en tiempo real

Características Principales

DAGs (Directed Acyclic Graphs)

Representación visual y programática de dependencias entre tareas

Reintentos Automáticos

Manejo de fallos con reintentos configurables y backoff exponencial

Multi-usuario

Control de acceso y roles para equipos colaborativos

Integraciones

Conectores para bases de datos, APIs, servicios en la nube

Historial

Trazabilidad completa de ejecuciones y cambios

Alertas

Notificaciones en caso de fallos o eventos importantes

Ventajas

Automatización

Eliminación de tareas manuales repetitivas y propensas a errores

Reducción de tiempo

Ejecución paralela de tareas independientes

Visibilidad

Dashboard centralizado para monitorear todos los workflows

Confiabilidad

Gestión automática de fallos y recuperación

Versionable

Workflows como código, controlables con Git

Escalabilidad

Crecimiento horizontal para manejar miles de workflows

Desventajas

Complejidad de instalación

Setup inicial complejo con múltiples componentes (broker, executor, base de datos)

Curva de aprendizaje

Requiere entender conceptos como DAGs, operators, y arquitectura

Overhead de infraestructura

Necesita recursos significativos para ejecutar la plataforma

Latencia

No ideal para procesamiento en tiempo real de baja latencia

Mantenimiento

Requiere dedicación para actualizaciones y gestión de dependencias

Debugging

Puede ser difícil diagnosticar problemas en workflows complejos

Apache Airflow

Plataforma Python para crear, programar y monitorear workflows

Historia

Creado por Airbnb en 2014, donado a Apache en 2016. Muy popular en la industria.

Lenguaje

Python nativo. Los workflows se definen en código Python

Comunidad

Comunidad grande, muchos integradores y extensiones disponibles

Casos de uso

ETL, pipelines de datos, machine learning, reportes automáticos

Fortalezas principales

Fácil de aprender para desarrolladores Python UI intuitivo y potente Excelente documentación Escalable a producción

Spring Cloud Data Flow (SCDF)

Plataforma Java para microservicios de procesamiento de datos

Origen

Desarrollado por Pivotal (ahora VMware). Parte del ecosistema Spring.

Arquitectura

Microservicios independientes con orquestación centralizada

Procesamiento

Soporta batch y stream processing con Cloud Stream

Casos de uso

Procesamiento de eventos, streaming de datos, cloud-native pipelines

Características destacadas

Diseñado para cloud (K8s, Cloud Foundry) Arquitectura basada en microservicios Ideal para streaming Integración con Spring ecosistema

Tutorial: Instalación de Apache Airflow

1 Requisitos Previos

• Python 3.8+

• pip (gestor de paquetes)

• Virtualenv recomendado

2 Crear entorno virtual

python -m venv airflow-env
source airflow-env/bin/activate # En Windows: airflow-env\Scripts\activate

3 Instalar Apache Airflow

pip install apache-airflow
pip install apache-airflow[postgres] # Para usar PostgreSQL

4 Inicializar base de datos

airflow db init

5 Crear usuario admin

airflow users create --username admin --password admin --firstname Admin --lastname User --role Admin --email admin@example.com

6 Iniciar servicios

airflow standalone # O en producción: airflow webserver & airflow scheduler

Tutorial: Primer DAG

Crear archivo: dags/hello_world_dag.py

from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime, timedelta

def print_hello():
    return 'Hello World!'

def print_goodbye():
    return 'Goodbye!'

default_args = {
    'owner': 'airflow',
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

dag = DAG(
    'hello_world_dag',
    default_args=default_args,
    schedule_interval='@daily',
    start_date=datetime(2024, 1, 1),
)

task1 = PythonOperator(
    task_id='say_hello',
    python_callable=print_hello,
    dag=dag,
)

task2 = PythonOperator(
    task_id='say_goodbye',
    python_callable=print_goodbye,
    dag=dag,
)

task1 >> task2

Ejecución y Monitoreo

Acceder a la UI

Navega a http://localhost:8080

Login con usuario: admin, password: admin

Ejecutar DAG

1. Busca "hello_world_dag" en la lista
2. Haz clic en el DAG para ver detalles
3. Haz clic en el botón play (►) para activar
4. Observa el estado en el Dashboard

Monitorear Ejecución

• Tree View: Visualiza estructura de DAG
• Graph View: Dependencias entre tareas
• Logs: Detalles de ejecución de cada tarea
• XCom: Compartir datos entre tareas
